## 通用設計風格 Prompt（每張畫面都要帶入）

```
設計風格：溫暖旅遊感，手繪風、插畫感，類似手作旅行手帳的質地
配色：暖色調為主（米白、奶油黃、珊瑚橘、淺木質色），避免冷色系科技感配色
字體：圓潤、易讀的無襯線字體，標題可帶手寫感
裝飾元素：可加入旅遊主題的小插畫點綴（行李箱、地圖、指南針、飛機等），但不干擾主要資訊閱讀
平台：Mobile-first（PWA），畫面寬度約 375–414px
```

## 1. Onboarding 問卷流程（Step-by-step）

```
設計一個 6 步驟的旅遊行程問卷頁面，Step-by-step 形式（一次只顯示一題，類似 Typeform）。

頁面結構：
- 頂部：進度條，分為 7 個區段，每個區段可點擊跳回對應的 step 進行修改
- 中間：當前 step 的問題與輸入元件，置中呈現，是頁面視覺焦點
- 底部：「上一步」「下一步」按鈕（選填題額外顯示「跳過」按鈕）

請分別設計以下 7 個 step 畫面：

Step 1（必填）：目的地輸入
- 文字輸入框，搭配下方 3-4 張熱門目的地建議卡片（卡片含插畫 + 地名）

Step 2（必填）：天數／夜數 + 出發/結束時間
- 日期區間選擇器（開始日期、結束日期）
- 選完日期後自動顯示計算出的「X 天 Y 夜」

Step 3（必填）：人數
- 數字增減器（- / + 按鈕），中間顯示當前人數，可搭配人物插畫圖示隨人數增加而增加

Step 4（必填）：旅遊性質
- 4 張插畫卡片並排或 2x2 排列：情侶出遊、家族出遊、獨旅、朋友出遊
- 每張卡片：上方插畫圖示 + 下方文字標籤，點擊整張卡片即選取（非小圓點 radio）
- 選中狀態需有明顯視覺變化（邊框高亮、背景色變化）

Step 5（選填）：預算範圍
- 滑桿（slider）選擇預算區間，右上角顯示「跳過」文字按鈕

Step 6（選填）：偏好風格
- 多選插畫卡片：文化探索、美食、戶外冒險、購物、放鬆度假等（4-6 個選項）
- 卡片可複選，選中狀態與 Step 4 一致的視覺邏輯
- 右上角顯示「跳過」文字按鈕

Step 7（選填）：其他要求
- 多行文字輸入框（textarea），placeholder 提示範例文字（如「其中一人使用輪椅，請避免需要爬坡的景點」、「團員中有素食者」）
- 輸入框下方可加入 2-3 個常見需求的快速標籤（如「無障礙需求」、「素食友善」、「避免人潮擁擠」），點擊可自動填入對應文字
- 右上角顯示「跳過」文字按鈕

完成所有 step 後，顯示一個「開始規劃行程」的 CTA 確認頁，列出使用者剛才填寫的所有答案摘要（包含其他要求若有填寫），提供「送出」按鈕。
```

## 2. AI 產生中等待畫面

```
設計一個 AI 行程產生中的等待頁面（全螢幕 loading 狀態，預期等待時間 10-30 秒）。

頁面結構：
- 畫面中央：旅遊主題插畫動畫的靜態關鍵幀（擇一）：
  選項 A：行李箱打包動畫（行李箱逐漸被填滿物品）
  選項 B：飛機飛行動畫（飛機沿著虛線航線飛過地圖）
- 動畫下方：提示文字，依進度變化（如「正在為你規劃行程...」→「正在挑選最佳景點...」→「即將完成...」）
- 底部：可選的小提示文字（如「精彩的旅程值得等待」），呼應溫暖旅遊感調性

避免使用制式的轉圈 loading icon 或灰色骨架屏，畫面整體應該讓使用者感覺「AI 正在用心準備驚喜」而非單純等待。
```

## 3. 行程主畫面（垂直時間軸）

```
設計一個行程詳細頁面，垂直時間軸呈現，手機畫面尺寸。

頁面結構（由上到下）：
- 頂部導覽列：行程標題（如「東京 3 天 2 夜」）、返回按鈕、「重新產生」按鈕（圖示）
- Day 切換 Tab：水平排列的 Tab（Day 1 / Day 2 / Day 3），當前選中的 Tab 需明顯高亮，點擊切換顯示對應天數內容
- 主要內容區：垂直時間軸
  - 左側：時間軸縱線，搭配時間標記（09:00、10:30 等）
  - 右側：行程節點卡片，依序垂直排列，卡片間有適當留白
  - 每張節點卡片包含：類別 icon（景點/餐廳/交通等不同圖示）、地點名稱（標題字級）、時間與時長、簡短描述（一行省略）
  - 卡片整體風格為圓角卡片，帶淺色陰影，呼應插畫手帳感
- 浮動的「+」按鈕（右下角），點擊新增節點

互動狀態示意（請額外設計以下 3 個變化版本）：

版本 A - 單擊節點後：節點旁彈出小型選單（編輯／刪除／複製 三個選項，圖示 + 文字）

版本 B - 長按節點觸發拖曳模式：該節點視覺上浮起（輕微放大、加深陰影），其他節點留出空隙顯示可放置位置，畫面上方出現提示文字「拖曳調整順序」

版本 C - 空白行程狀態（該天尚無任何節點，或新行程剛建立時）：時間軸區域顯示插畫引導圖（如空白地圖或行李箱），搭配文字「這天還沒有安排，點擊 + 開始規劃」

版本 D - 「換一個」備選選項展示：
- 被替換的節點進入 loading 狀態（骨架屏，寬度與原節點相同）
- AI 回傳後，原節點位置展開顯示 3 張備選卡片（垂直列出）
- 每張備選卡片包含：類別 icon、地點名稱、簡短描述、預計時長
- 卡片底部有「選擇」按鈕（主色）
- 整組卡片頂部有「取消」文字按鈕，點擊收合回原本節點
```

## 5. 分享管理畫面

```
設計一個行程分享設定頁面（從行程主畫面的分享按鈕進入），mobile 畫面尺寸。

頁面結構（由上到下）：
- 頂部導覽列：「分享行程」標題，返回按鈕
- 分享連結區塊：
  - 標題「分享連結」
  - 兩個連結 Card，各自顯示：
    - 權限標籤（「唯讀」/「可編輯」）+ 權限說明文字（如「對方只能檢視，不可修改」）
    - 連結預覽（截斷顯示）
    - 「複製連結」按鈕、「停用」切換開關
  - 底部「+ 建立新連結」按鈕（點擊後選擇唯讀或可編輯）
- 匯出 PDF 區塊：
  - 標題「匯出行程」
  - 一張 Card，說明「產生完整行程 PDF 文件」
  - 「匯出 PDF」按鈕，點擊後顯示 loading 狀態（PDF 產生中）

整體卡片風格維持溫暖旅遊感，圓角卡片 + 淺色陰影，與行程主畫面一致。
```

## 6. 行程列表頁

```
設計一個使用者的行程列表頁（App 首頁，登入後看到的第一個畫面）。

頁面結構：
- 頂部：App logo（手繪風格）、使用者頭像（右上角），頭像可點擊進入個人設定
- 搜尋列（可選）：搜尋已存行程
- 主要內容：
  - 「即將到來的行程」區塊：出發日期未到的行程卡片（2-3 張）
  - 「過去的行程」區塊：已結束的行程卡片列表
  - 每張行程卡片包含：旅遊地點插畫/封面圖、行程標題、日期區間、人數、旅遊性質標籤
- 右下角浮動「+」按鈕：點擊進入 Onboarding 問卷，開始規劃新行程

空白狀態（尚未有任何行程）：
  畫面中央顯示旅遊主題插畫（如空白地圖 + 行李箱），搭配引導文字「還沒有行程，點擊 + 開始規劃你的第一趟旅程」

共同編輯提示（當某個行程有其他人正在編輯時）：
  行程卡片上顯示小型 badge「有人正在編輯」，點進去後在行程主畫面頂部顯示警告 banner
```


```
設計一個行程節點編輯彈窗（Modal），從畫面底部滑出（bottom sheet 形式，符合 mobile 操作習慣）。

內容包含：
- 頂部：拖曳把手（小橫條），標題「編輯行程」，右上角關閉按鈕
- 表單欄位：
  - 地點名稱（文字輸入）
  - 類別選擇（icon 選擇器：景點/餐廳/交通/住宿/其他）
  - 時間（時間選擇器）
  - 時長（數字輸入，單位分鐘，或滑桿選擇常見區間）
  - 地點地址（文字輸入，可選搭配地圖選點）
  - 備註描述（多行文字輸入）
- 底部：「儲存」主要按鈕（與整體暖色調一致）、「取消」次要按鈕
```
